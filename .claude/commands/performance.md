전기짱(mycbt) 사이트의 성능을 점검하고 최적화 방안을 제시해줘.

## 점검 항목

### 1. 번들 크기
- `npx next build`로 각 페이지 번들 크기 확인
- 불필요하게 큰 페이지 식별
- lazy loading이 필요한 컴포넌트 확인

### 2. 데이터 페칭
- 불필요한 폴링 확인 (현재 10초 폴링이 필요한지)
- API 응답에 불필요한 데이터가 포함되어 있지 않은지
- N+1 쿼리 패턴 확인 (Prisma include vs 별도 쿼리)

### 3. 렌더링 최적화
- 서버 컴포넌트 vs 클라이언트 컴포넌트 적절성
- 'use client' 범위가 최소한인지
- 불필요한 리렌더링 유발 패턴

### 4. 캐싱
- API 응답의 Cache-Control 헤더 적절성
- 정적으로 생성 가능한 페이지 확인
- Prisma 쿼리 최적화 (select로 필요한 필드만)

### 5. 이미지/폰트
- next/image 사용 여부
- 폰트 로딩 전략 (next/font 사용 여부)

우선순위별로 정리하고, 즉시 적용 가능한 것과 장기 개선 사항을 구분해줘.
