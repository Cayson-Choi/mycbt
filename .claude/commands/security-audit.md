전기짱(mycbt) 코드베이스의 보안 취약점을 점검해줘.

## 점검 항목

### 1. 인증/인가
- 모든 API 라우트에 세션 검증이 있는지
- 관리자 API에 isAdmin 검증이 있는지
- 사용자 데이터 접근 시 userId 소유권 확인이 있는지

### 2. 데이터 노출
- 정답(answer)이 시험 중 프론트로 내려가지 않는지
- 다른 사용자의 데이터에 접근 가능한 경로가 없는지
- 에러 메시지에 내부 정보가 노출되지 않는지

### 3. 입력 검증
- API에서 입력값 검증 (타입, 범위, 형식)
- SQL Injection 방어 (Prisma 사용하므로 대부분 안전하지만 raw query 확인)
- XSS 방어 (사용자 입력이 그대로 렌더링되는 곳)

### 4. 미들웨어
- 보호된 경로가 빠짐없이 미들웨어에 포함되어 있는지
- 공개 API 목록이 최소한인지

### 5. 환경변수
- .env.local이 .gitignore에 포함되어 있는지
- NEXT_PUBLIC_ 접두사가 불필요하게 붙은 변수가 없는지

취약점 발견 시 심각도(높음/중간/낮음)와 수정 방법을 제시해줘.
