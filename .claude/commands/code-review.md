최근 변경된 코드를 리뷰해줘.

## 절차
1. `git diff HEAD~3` 또는 사용자가 지정한 범위의 변경사항 확인
2. 다음 관점에서 리뷰:

### 코드 품질
- 불필요한 코드, 중복 코드
- 에러 처리 누락
- 타입 안정성 (any 남용 여부)

### 보안
- API에서 인증/인가 검증 누락
- 민감 데이터 노출

### 성능
- 불필요한 리렌더링 유발
- N+1 쿼리 패턴
- 큰 번들 사이즈 유발

### 일관성
- snake_case vs camelCase 혼용
- 기존 코드 패턴과의 일관성

문제 발견 시 파일:줄번호와 함께 구체적 수정 제안을 해줘.
